# AFO Client Site Pipeline v0.1.3 patch notes

Canonical source seed:

`workers/afo-client-site-pipeline-mcp/src/worker-v0.1.3-prep.js`

## Confirmed runtime bug

The live v0.1.2 Client Site Pipeline export includes duplicate homepage artifacts for the CCS project.

Project:

`clientsite_ca7e4722-39fa-4b18-afcb-f403371bca2f`

Current bad export behavior:

- `export_site_bundle` returns `page_count: 5`.
- It includes the older landing artifact `artifact_d59044fb-6a2e-4504-868c-b9c0b0bddadd`.
- It also includes the newer landing artifact `artifact_7d56a370-c96a-40f8-9de4-a966d7f36e5e`.
- The older landing artifact is the wrong commercial-cleaning/janitorial version.
- The newer landing artifact is the correct construction/remodeling version.

Site Preview MCP currently has the CCS preview record at 4 pages, so the preview side is healthy. The bug is in pipeline export selection.

## Required v0.1.3 behavior

`export_site_bundle` must be latest-only by default.

Selection rule:

```txt
For each canonical route:
  group page artifacts by route
  landing_page_draft route is /
  content_page_draft route is data.route or /articles/{slug}
  prefer current/approved/reviewed/published/ready status
  then prefer newest timestamp
  then prefer newest artifact id as stable tiebreak
  select exactly one page per route
  put older same-route artifacts in excluded_artifacts with reason superseded_by_latest
```

Expected CCS result:

```txt
page_count: 4
selected homepage: artifact_7d56a370-c96a-40f8-9de4-a966d7f36e5e
excluded homepage: artifact_d59044fb-6a2e-4504-868c-b9c0b0bddadd
excluded reason: superseded_by_latest
```

## New tool required

Add `create_sample_site_from_profile` as a one-call structured-profile workflow:

```txt
business_profile
→ create_client_site_project
→ build/refine intake profile
→ generate_site_plan
→ generate_landing_page_draft
→ create_content_page_drafts
→ build_sample_site_bundle
→ run_client_site_review
→ export_site_bundle(mode=latest_only)
→ create_publisher_manifest
→ create_catalog_payload
→ return preview_url + IDs + review/export/catalog metadata
```

Preview publication remains delegated to Site Preview MCP. URL parsing remains v0.2 and should orchestrate existing parse tools rather than rebuilding them.
