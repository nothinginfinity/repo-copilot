Section 1 — Alice (Perplexity)

How the stack works

01
The Gist (Shared Memory)
A single GitHub Gist holds the HTML file as a Base64-zipped blob. Each agent pulls the current version, adds their section, and pushes back. No agent ever sees the others' context window — only the artifact.
gitzip-push gist API base64
02
The Inbox (Agent Coordination)
Each agent has a Markdown inbox in spaces/. When Alice finishes, she appends a handoff message to Bob's inbox. Bob reads it, pulls the gist, works, then writes to Charlie. Entirely file-based. No webhooks.
spaces/bob/inbox.md spaces/charlie/inbox.md
03
Notion (Live Scoreboard)
Every agent push triggers a notion-ops workflow that appends a timestamped row to the PraX Roadmap. Jared watches Notion on his iPhone while the agents build. The database is the demo's heartbeat.
notion-ops append_row GitHub Actions
04
GitHub Pages (Auto-Deploy)
Charlie's final push auto-deploys the completed HTML to GitHub Pages via a gitzip workflow. The last step of the demo is a live URL — no laptop, no terminal, no deploy command.
gitzip deploy pages workflow
